# Project Requirement Summary

## 1. 项目一句话定义
一个本地面试复习 App：从本地 Markdown 文件自动解析题目，用户答题后对比参考答案，打标记追踪掌握情况。

## 2. 目标用户
开发者本人，本机单用户使用，笔记本浏览器访问。

## 3. 核心问题
面试资料散落在多个 Markdown 文件里，复习时需要手动翻文件，无法追踪哪些题掌握了、哪些还需要再刷。

## 4. 为什么现在做
准备面试中，需要高频复习，当前纯翻文件效率低。

## 5. 核心场景
1. 选择分类（Java基础、Redis、算法题等）+ 模式（顺序/随机）
2. 看题 → 文本框写思路
3. 点"查看答案"，参考答案在同页滑入展开
4. 打标记：✅ 已掌握 / ⚠ 不熟 / 🔁 待复习
5. 进入下一题
6. 查看统计（各分类掌握率）和答题历史

## 6. 输入 / 输出
- 输入：本地 `面试/` 目录下的 `.md` 文件（八股文按 `##` 拆题，算法题每文件一题）
- 输出：题目展示、参考答案对比、标记状态持久化、进度统计

## 7. 成功标准
- 能从 Markdown 文件自动解析出所有题目，无需手动导入
- 答题 → 查看答案 → 打标记 → 下一题，整个流程无需跳页
- 标记记录持久化，重启后保留
- 统计页能看到各分类掌握率和答题历史

## 8. 第一阶段目标
完整跑通：选题 → 答题 → 打标记 → 统计，所有题目从本地 MD 文件实时读取。

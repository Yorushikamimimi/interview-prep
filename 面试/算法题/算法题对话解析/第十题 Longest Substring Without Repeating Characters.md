题目：Longest Substring Without Repeating Characters  
示例：

- 输入："abcabcbb" → 输出：3  
    因为最长的不含重复字符的子串是 "abc"
    
- 输入："bbbbb" → 输出：1  
    因为最长的是 "b"
    
- 输入："pwwkew" → 输出：3  
    因为最长的是 "wke"，注意 "pwke" 不是连续子串，不能算
    

结论：这题大概率属于 字符串 + sliding window（滑动窗口） 题型候选，本质是在“连续区间”里动态维护一个合法窗口。

## 已知事实

1. 题目本质  
    这题要找的是：  
    在一个字符串里，找到一个连续子串，要求这个子串中没有重复字符，并返回它的最大长度。
    
2. 核心思想候选  
    这类题常见方向有两个：
    

- 暴力枚举（brute force）：枚举所有子串，再判断有没有重复
    
- 滑动窗口（sliding window）：维护一个连续区间，边移动边保证“窗口内无重复”
    

4. 你必须先抓住的关键点
    

- 题目求的是 substring，不是 subsequence
    
- substring（子串）：连续的一段
    
- subsequence（子序列）：可以不连续地选
    

6. 常用数据结构候选
    

- HashSet：用来判断某个字符是否已经出现
    
- HashMap：记录某个字符最近出现的位置，方便更快移动左边界

2. baseline 可能是双重循环枚举子串；optimal 大概率是滑动窗口。
    
3. 真正难点不在代码，而在于：

## 关键教学：窗口坏了以后该怎么修

- 移出 c
    
- 窗口里还是有旧 a
    
- 仍然重复
    

### 第三次移动

- 移出 d
    
- 窗口里还是有旧 a
    
- 仍然重复
    

### 第四次移动

- 移出旧 a

题目：Longest Substring Without Repeating Characters  
示例：

- "abcabcbb" → 3，最长无重复子串是 "abc"
    
- "bbbbb" → 1，最长无重复子串是 "b"
    
- "pwwkew" → 3，最长无重复子串是 "wke"
    

结论：这题属于 字符串 + Sliding Window（滑动窗口，维护一个连续合法区间）。  
baseline 是 双层枚举 + HashSet 判重，optimal 是 双指针滑动窗口 + HashSet/HashMap 维护窗口状态。

---

## 已知事实

### 1）题目本质

在字符串中找一个连续子串 substring，要求其中没有重复字符，返回这个子串的最大长度。

### 2）核心术语

- Substring：子串，必须连续
    
- Sliding Window：滑动窗口，在字符串上维护一个连续区间，并动态调整左右边界
    
- HashSet：哈希集合，用来快速判断某个字符是否已在当前窗口中
    
- HashMap：哈希映射，记录字符最近出现位置，可进一步优化左指针跳跃
    

### 3）baseline 思路

外层枚举起点，内层向右扩张终点，用 HashSet 判断当前子串内是否重复。  
一旦重复，就结束本轮内层循环，换下一个起点。

### 4）optimal 思路

维护一个当前合法窗口：

- 右指针负责扩张窗口，尝试加入新字符
    
- 如果新字符导致重复，左指针持续右移，并同步删除移出窗口的字符
    
- 直到窗口重新合法
    
- 每次窗口合法后更新最大长度
    

---

## 合理推测

### 1）为什么 baseline 会慢

因为有很多重复劳动。  
比如从某个起点扫过的字符，换下一个起点时又会重新扫一遍。

### 2）为什么滑动窗口更优

因为它不是每次从头重来，而是：

- 右边继续向前看
    
- 左边按需缩小窗口  
    这样每个字符最多被左右指针处理有限次。
    

### 3）最容易错的点

不是代码，而是概念：

- HashSet 维护的是当前窗口内的字符，不是全局出现过的字符
    
- 左指针右移时，必须把移出窗口的字符从集合中删除
    
- 左指针不是固定移动一次，而是持续移动到窗口重新合法为止

# 最终结果

## 一、baseline 方案

### Core Idea

双层循环枚举所有可能起点，从每个起点向右扩展，用 HashSet 判断是否重复。

### 主流程

1. 外层枚举左端点 i
    
2. 内层从 i 开始往右走
    
3. 用 HashSet 记录当前子串中已出现字符
    
4. 如果当前字符不重复，加入集合并更新最大长度
    
5. 如果重复，结束当前内层循环
    

### 复杂度

- 时间复杂度：O(n^2)  
    这里 n 是字符串长度
    
- 空间复杂度：O(k)  
    k 是当前子串中不同字符个数，最坏可视作 O(n)
    

---

## 二、optimal 方案

### Core Idea

用滑动窗口维护一个“无重复字符”的合法连续区间。

### 主流程

1. 用两个指针 left、right 表示当前窗口
    
2. 右指针不断尝试加入新字符
    
3. 若新字符已存在于当前窗口中，则窗口失效
    
4. 左指针持续右移，同时从 HashSet 删除移出的字符
    
5. 直到新字符不再重复，窗口恢复合法
    
6. 用当前窗口长度更新答案
    

### 伪代码骨架

set = 空集合

left = 0

ans = 0

  

for right in [0 .. n-1]:

    while s[right] 在 set 中:

        从 set 删除 s[left]

        left += 1

  

    把 s[right] 加入 set

    ans = max(ans, right - left + 1)

  

### 复杂度

- 时间复杂度：O(n)  
    因为左右指针都只单调向右移动，每个字符最多被加入和删除一次
    
- 空间复杂度：O(k)  
    最坏可视作 O(n)

# 面试可复述版本

这题可以先用双层枚举加 HashSet 做 baseline：外层枚举起点，内层向右扩展，一旦遇到重复字符就停止。  
更优做法是滑动窗口。用两个指针维护当前无重复子串，右指针负责扩张窗口，左指针在出现重复时持续收缩窗口，并同步更新集合，直到窗口重新合法。  
这样可以把重复扫描降到最低，时间复杂度从 O(n^2) 优化到 O(n)。

# 一句话题型总结

这题属于 字符串 + 滑动窗口，本质是维护一个无重复字符的合法连续区间。

# 一句话核心 insight

这题最关键的突破点是：出现重复时不要整段重来，而是通过移动左指针收缩窗口，直到窗口重新合法。

题目：Reverse Linked List（反转链表）

示例：  
输入：1 -> 2 -> 3 -> 4 -> 5 -> null  
输出：5 -> 4 -> 3 -> 2 -> 1 -> null

再比如：  
输入：1 -> 2 -> null  
输出：2 -> 1 -> null

---

## 结论

这题大概属于 Linked List（链表）操作题，更具体一点，是 Pointer Manipulation（指针操作） 题。  
本质是：把链表中每个节点的指向方向反过来。

---

## 已知事实

1. 题目本质  
    不是改节点的值，而是改 next pointer（下一个指针） 的指向。
    
2. 核心数据结构  
    用的是 Singly Linked List（单链表），每个节点通常只有：
    

- val
    
- next
    

4. 核心思想候选
    

- Iterative（迭代）：一边遍历，一边修改指针方向
    
- Recursive（递归）：先反转后半段，再回头改当前节点指向
    

6. 这题最关键的风险  
    链表一旦改指针，后面的节点可能丢失。  
    所以通常要先想：改当前指针之前，怎么保存后继节点。
    

---

## 合理推测

1. 这题大概率是想考你对链表指针的控制能力，而不是复杂算法。
    
2. 面试里这题经常被当成链表基础题，重点看你是否真的理解：
    

- 当前节点
    
- 前一个节点
    
- 下一个节点  
    三者关系。
    

4. 这题的最优解通常不复杂，重点反而是别写错、别断链。
    

---

## 需要验证

你现在需要先确认这几个理解点：

1. 你是否清楚单链表的节点结构是什么。
    
2. 你是否明白“反转”不是把值倒过来，而是把 next 的方向反过来。
    
3. 你是否能看出：如果直接改 cur.next，可能会把后面的链表弄丢。
    

---

## 术语解释

- Linked List（链表）：节点通过指针串起来的线性结构，不像数组那样连续存储。
    
- Pointer（指针）：这里可以简单理解为“指向下一个节点的引用”。
    
- Iterative（迭代）：用循环一步步处理。
    
- Recursive（递归）：函数调用自己，把问题拆小再返回。

题目：Reverse Linked List

示例：  
输入：1 -> 2 -> 3 -> 4 -> 5 -> null  
输出：5 -> 4 -> 3 -> 2 -> 1 -> null

---

## 结论

这题属于 Linked List（链表）+ Pointer Manipulation（指针操作）。  
本质是：把每个节点的 next 指针方向反过来，最终得到一个新的链表头。

---

## 已知事实

1. 这题反转的是 指针方向，不是节点里的值。
    
2. 单链表一旦改了 cur.next，如果没提前保存后继节点，就会 断链。
    
3. 标准做法是维护三个指针：
    

- prev：已经反转好的前半段链表头
    
- cur：当前正在处理的节点
    
- next：临时保存 cur 原本的下一个节点
    

5. 循环结束条件是 cur == null，因为链表走到尾部后，next 会变成 null。
    

---

## 合理推测

1. 这题重点不是难算法，而是看你是否真的理解链表指针如何流转。
    
2. 面试里常用来考察你是否能稳定说清楚：
    

- 为什么要先保存 next
    
- 为什么返回的是 prev
    
- 为什么时间复杂度是 O(n)、空间复杂度是 O(1)
    

---

## 需要验证

1. 空链表时是否成立：成立，直接返回 null。
    
2. 只有一个节点时是否成立：成立，反转后还是自己。
    
3. 是否理解 prev / cur / next 三者分工：这是这题真正的关键。
    

---

## 核心术语

- Linked List（链表）：节点通过指针串联的线性结构，不是连续内存。
    
- Pointer（指针）：这里可理解为“指向下一个节点的引用”。
    
- Iterative（迭代）：用循环逐步处理。
    
- Recursive（递归）：函数调用自己，把问题拆小后再回溯处理。
    

---

## baseline solution（朴素解）

可以先把所有节点放进 array / stack（数组 / 栈），然后再倒序重新连接。

### 思路

1. 遍历链表，把节点依次存起来
    
2. 倒序取出并重新连接
    
3. 最后把尾节点指向 null
    

### 问题

- 需要额外 O(n) 空间
    
- 虽然能做，但不是最优
    

---

## optimal solution（最优解）

使用 三个指针原地反转：

- prev = null
    
- cur = head
    

每一轮做四步：

1. next = cur.next，先保存后继节点
    
2. cur.next = prev，把当前节点反过来
    
3. prev = cur，让 prev 前进一步
    
4. cur = next，让 cur 处理下一个节点
    

循环直到 cur == null，最后返回 prev。

---

## 伪代码骨架

prev = null

cur = head

  

while cur != null:

    next = cur.next

    cur.next = prev

    prev = cur

    cur = next

  

return prev

  

---

## 为什么这样设计

### 1. 为什么要先保存 next

因为如果直接执行 cur.next = prev，原来通往后面链表的路径就断了，后续节点会丢失。

### 2. 为什么返回 prev

因为循环结束时：

- cur = null
    
- prev 正好指向反转后链表的新头节点
    

所以必须返回 prev，不能返回 cur。

---

## 复杂度分析

设链表长度为 n。

- 时间复杂度：O(n)  
    每个节点只访问和修改一次。
    
- 空间复杂度：O(1)  
    只用了有限个指针变量，没有额外开数组或栈。
    

---

## 容易错的点

1. 没先保存 next 就改 cur.next  
    会直接断链。
    
2. 把 prev 和 cur 的推进顺序弄乱  
    会导致指针关系错乱。
    
3. 最后返回 cur 而不是 prev  
    循环结束时 cur 已经是 null。
    
4. 把这题理解成数组反转  
    链表没有下标，结束条件看的是是否走到 null。
    

---

## recursive（递归）做法的 trade-off

递归也能做，但和迭代相比：

### 优点

- 写法有时更简洁
    
- 对理解“递归反转链表”有帮助
    

### 缺点

- 需要递归栈，空间复杂度变成 O(n)
    
- 边界更绕，面试里更容易说乱
    

所以一般面试默认优先讲 迭代版。

---

## 面试可复述版本

这题是链表指针操作题，核心是原地反转单链表。做法是维护 prev、cur、next 三个指针：每轮先保存 cur 原本的下一个节点 next，防止断链；再把 cur.next 指向 prev 完成局部反转；然后同时推进 prev 和 cur。当 cur == null 时说明所有节点都处理完了，此时 prev 指向反转后链表的新头节点，返回 prev 即可。时间复杂度是 O(n)，空间复杂度是 O(1)。

---

## 一句话题型总结

- 这题属于 Linked List + Pointer Manipulation，本质是通过修改每个节点的 next 指针来原地反转链表。
    

## 一句话核心 insight

- 这题最关键的突破点是：每次改指针前必须先保存原来的下一个节点，否则会断链。

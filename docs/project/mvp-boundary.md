# MVP Boundary

## MVP 必做
- 后端自动扫描 `面试/` 目录，解析 Markdown 题目（八股文 + 算法题两种格式）
- 分类筛选 + 顺序/随机两种出题模式
- 答题文本区（非代码编辑器，普通 textarea）
- 同页展开参考答案，Markdown 渲染 + 代码语法高亮
- 三状态自我标记：已掌握 / 不熟 / 待复习，持久化到 SQLite
- 进度条（本次练习 X / N 题）
- 统计页：各分类掌握率 + 答题历史列表
- 待复习快捷入口（只显示标记为「待复习」的题）

## 明确不做（本轮）
- 用户登录 / 权限系统
- 多设备同步
- AI 评分（Claude API 对比答案）
- 题目编辑 / 在页面内修改 Markdown 原文
- 艾宾浩斯复习计划 / 定时提醒
- 答题草稿自动保存（中途退出不保留输入内容）
- 搜索功能
- 部署上线（仅本地运行）
- Docker / systemd 部署

## 后续可能做，但本轮不做
- AI 评分：写完答案后调用 API 给出对比评价
- 复习计划：基于标记状态自动排期
- 关键词搜索题目
- 多设备同步（需要服务端部署）

## 容易过度设计的点
- MD 解析不需要 100% 兼容所有语法，只需支持现有文件格式（`##` 标题拆题 + 代码块）
- 统计不需要复杂图表库，CSS SVG 够用
- 不需要 Redis、MQ、搜索引擎、管理后台

## 本轮范围声明
- 一轮只推进一个模块
- 任何会新增 Redis / MQ / Search / Admin / 多服务拆分的需求，先回到技术决策阶段

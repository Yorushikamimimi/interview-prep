题目：Maximum Depth of Binary Tree

给你一棵二叉树 root，返回它的最大深度。

示例：

输入：root = [3,9,20,null,null,15,7]

输出：3

  

对应的树大概是：

     3

     / \

    9  20

      /  \

     15   7

  

---

## 结论

这题大概是 Binary Tree Traversal（二叉树遍历） / DFS（Depth-First Search，深度优先搜索） 的入门题，也可以用 BFS（Breadth-First Search，广度优先搜索） 做。

---

## 已知事实

1. 题目本质
    

- 题目要你求的是：从根节点到最深叶子节点，最长有多少层。
    

3. 核心思想候选
    

- 可以用 DFS：一路往下递归，分别求左子树和右子树的深度。
    
- 也可以用 BFS：一层一层遍历，最后统计一共多少层。
    

5. 这题最常见的数据结构 / 方法
    

- 递归 recursion：函数调用自己，用来处理树这种天然分层结构。
    
- 队列 queue：BFS 按层遍历时常用。
    

7. 这题是非常标准的树基础题
    

- 重点不是技巧，而是你能不能把“树的高度 / 深度”这个概念和遍历方式对应起来。

## 核心术语

- DFS（Depth-First Search）：先一路往深处走到底，再回头处理别的分支。
    
- BFS（Breadth-First Search）：一层一层往下扫。
    
- 递归 recursion：函数把大问题拆成更小的同类子问题。
    
- depth：当前节点处在第几层。
    
- height：以当前节点为根，这棵子树有多高。
    

这题里你可以把它理解成：求整棵树的高度。

---

## 先给你最小理解框架

这题最自然的想法是：

- 一棵树的最大深度  
    = max(左子树最大深度, 右子树最大深度) + 1
    

因为：

- 左边最深能到多少层
    
- 右边最深能到多少层
    
- 当前根节点自己也要算一层

这题是二叉树 DFS / 递归的经典入门题，本质是：当前树的最大深度 = 左子树最大深度和右子树最大深度中的较大值 + 1。

## 已知事实

1. 题目要解决什么
    

- 求一棵二叉树从根节点到最深叶子节点一共有多少层。
    

3. 核心思想
    

- 把大问题拆成子问题：
    

- 左子树最大深度是多少
    
- 右子树最大深度是多少
    

当前节点答案就是：  
max(leftDepth, rightDepth) + 1

-   
    

3. 边界
    

- 如果节点是 null，深度就是 0。
    
- 如果是叶子节点，深度就是 1。
    

5. 核心术语
    

- DFS (Depth-First Search)：沿一条分支不断往下搜索。
    
- Recursion（递归）：函数自己处理更小规模的同类问题。
    
- BFS (Breadth-First Search)：一层一层遍历树。

## Baseline Solution

### 方案一：DFS 递归

思路：

- 如果当前节点为空，返回 0
    
- 递归求左子树深度
    
- 递归求右子树深度
    
- 返回两者较大值 + 1
    

伪代码：

maxDepth(root):

    if root == null:

        return 0

  

    left = maxDepth(root.left)

    right = maxDepth(root.right)

  

    return max(left, right) + 1

  

---

## Optimal Solution

### 方案二：BFS 层序遍历

思路：

- 用队列一层一层遍历
    
- 每遍历完一层，depth + 1
    
- 最后返回层数
    

伪代码：

maxDepth(root):

    if root == null:

        return 0

  

    queue = [root]

    depth = 0

  

    while queue is not empty:

        size = queue.size()

  

        repeat size times:

            node = queue.pop()

            if node.left != null:

                queue.push(node.left)

            if node.right != null:

                queue.push(node.right)

  

        depth += 1

  

    return depth

  

---

## 为什么 baseline 就已经很好

这题其实 DFS 递归就是最主流、最自然的最优解。  
严格来说，这题不是那种“先很差，再大优化”的题，而是：

- DFS：代码最简洁，最符合树的定义
    
- BFS：同样是 O(n)，只是思路不同
    

所以这题更像是：

- 两种等价解法
    
- 各有适用场景
    

---

## 复杂度分析

设树中节点总数为 n。

### DFS

- Time Complexity: O(n)
    

- 每个节点访问一次
    

- Space Complexity: O(h)
    

- h 是树高
    
- 递归栈深度取决于树高
    
- 最坏情况链状树时是 O(n)
    
- 平衡树时大约是 O(log n)
    

### BFS

- Time Complexity: O(n)
    
- Space Complexity: O(w)
    

- w 是某一层的最大节点数
    
- 最坏情况可到 O(n)

## 容易错的点

1. 把空节点深度写错
    

- null 应该返回 0，不是 1
    

3. 混淆“节点数”和“边数”
    

- LeetCode 这题算的是层数 / 节点层级数
    

5. 忘了当前节点自己这一层
    

- 所以最后一定要 +1
    

7. 把最大深度写成左右相加
    

- 这是错的
    
- 应该取 max，不是 left + right

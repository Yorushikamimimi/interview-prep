# 集合

---

## 1. List / Set / Map

- **List:** 有序，可重复（ArrayList, LinkedList）
- **Set:** 无重复（HashSet, TreeSet）
- **Map:** 键值对，key 不重复（HashMap, TreeMap）
- Map 和 Collection 是两套体系

---

## 2. ArrayList vs LinkedList

| 对比 | ArrayList | LinkedList |
|------|-----------|------------|
| 底层 | 动态数组 | 双向链表 |
| 查询 | O(1) 随机访问 | O(n) 遍历 |
| 增删 | 中间插入需移动元素 | 节点增删方便 |
| 实际 | 大多数场景优先 | 仅头尾频繁操作时考虑 |

### ArrayList 扩容

容量不够时扩到 1.5 倍，本质是新建数组 + 拷贝数据。预估数据量时初始化指定容量更好。

---

## 3. HashMap

**答:** 底层 数组 + 链表 + 红黑树，通过 hash 定位桶，链表处理冲突，冲突严重时树化。

### put 流程

算 hash → 找桶 → 空则直接放 → 非空比较 key → 相同覆盖 / 不同挂链表或树 → 超过负载因子则扩容

### 1.7 vs 1.8

| 版本 | 冲突处理 | 插入方式 |
|------|---------|---------|
| 1.7 | 数组+链表 | 头插法 |
| 1.8 | 数组+链表+红黑树 | 尾插法 |

### 树化条件

- 链表长度 >= 8 **且** 数组容量 >= 64 才树化
- 容量 < 64 时优先扩容（冲突多可能是表太小）

### 为什么线程不安全

无同步控制，并发 put/resize 可能数据覆盖、丢失、结构异常。

---

## 4. ConcurrentHashMap

**答:** 线程安全的 HashMap，JDK 1.8 基于 CAS + synchronized，锁粒度比 Hashtable 细得多。

| 版本 | 实现 |
|------|------|
| 1.7 | Segment 分段锁 |
| 1.8 | CAS + synchronized，数组+链表+红黑树 |

### vs synchronizedMap

synchronizedMap 整体加锁，ConcurrentHashMap 锁粒度更细，并发性能更好。

---

## 5. HashSet

**答:** 底层基于 HashMap，元素存为 key，value 是固定占位对象，天然去重。

- 自定义对象放 HashSet 去重失败 → 没有正确重写 equals() 和 hashCode()

---

## 6. fail-fast

**答:** 遍历集合时检测到结构被修改 → 抛 ConcurrentModificationException。通过 modCount 检测。

- 安全删除：用 Iterator.remove()，不要在 for-each 里直接 list.remove()

---

## 7. 其他集合

- **CopyOnWriteArrayList:** 写时复制，读不加锁，适合读多写少
- **LinkedHashMap:** HashMap + 双向链表，保持插入/访问顺序，可做 LRU
- **TreeMap:** 红黑树，按 key 排序

---

## 8. 选型速查

| 需求 | 选择 |
|------|------|
| 普通 KV 查找 | HashMap |
| 并发 KV | ConcurrentHashMap |
| 保持顺序 | LinkedHashMap |
| 按 key 排序 | TreeMap |
| 普通列表 | ArrayList |
| 并发列表(读多写少) | CopyOnWriteArrayList |
| 去重 | HashSet |

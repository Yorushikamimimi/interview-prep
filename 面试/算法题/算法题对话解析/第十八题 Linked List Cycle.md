题目：Linked List Cycle

示例：  
给你一个单链表的头节点 head ，判断链表中是否有环。  
这里的“有环”是指：某个节点的 next 最终又指回了前面已经访问过的节点。

例如：

3 -> 2 -> 0 -> -4

     ^         |

     |_________|

  

这个链表就有环。

再比如：

1 -> 2 -> null

  

这个链表就没有环。

---

## 先给结论

这题大概属于：Linked List + Cycle Detection（链表判环） 题型，常见候选思路是 Hash Set 和 Fast & Slow Pointers（快慢指针）。

---

## 已知事实

1. 题目本质  
    这题不是让你找环的长度，也不是让你找入环点，最基础目标只是：判断有没有环。
    
2. 核心数据结构 / 思想
    

- Hash Set：哈希集合，用来记录“这个节点我之前见过没有”。
    
- Fast & Slow Pointers：快慢指针，两个指针以不同速度前进，如果有环，最终会相遇。
    

4. 为什么这题适合链表特殊技巧  
    链表没有下标，不能像数组那样随机访问，所以很多时候要靠“指针关系”来做判断。
    
5. 这题的标准默认前提
    

- 可能是空链表
    
- 可能只有一个节点
    
- 环可能很短，也可能很长
    
- 如果无环，链表最终会走到 null
    

---

## 合理推测

1. 这题通常会先想到一个更直观的 baseline solution（朴素解）：遍历时把访问过的节点存起来。
    
2. 更优的 optimal solution（最优解） 往往是快慢指针，因为它能把额外空间降下来。
    
3. 后续很多相关题，比如：
    

- 找环的起点
    
- 找环长度  
    都是从这题继续延伸出来的。

## 一句话题型总结

这题属于 链表判环题，本质是判断指针遍历过程中是否会进入循环路径。

---

## 一句话核心 insight

这题最关键的突破点是：用快慢指针制造速度差，如果有环，快指针最终一定会追上慢指针。

---

## baseline 方案

### Core Idea

遍历链表，同时用 Hash Set 记录访问过的节点；如果某个节点再次出现，说明有环。

### 伪代码骨架

set = empty set

cur = head

  

while cur != null:

    if cur in set:

        return true

    add cur to set

    cur = cur.next

  

return false

  

### 复杂度

- Time Complexity：O(n)
    
- Space Complexity：O(n)
    

---

## optimal 方案

### Core Idea

使用快慢指针：

- slow 一次一步
    
- fast 一次两步
    

如果链表有环，两者最终会相遇；如果无环，fast 会先到 null。

### 伪代码骨架

slow = head

fast = head

  

while fast != null and fast.next != null:

    slow = slow.next

    fast = fast.next.next

  

    if slow == fast:

        return true

  

return false

  

### 复杂度

- Time Complexity：O(n)
    
- Space Complexity：O(1)
    

---

## 为什么 baseline 不是最优

不是因为它时间特别差，而是因为它用了额外的 Hash Set。  
而快慢指针在同样保持 O(n) 时间的情况下，把空间降到了 O(1)。

---

## 容易错的点

1. 判断的是节点是否重复，不是值是否重复。
    
2. while 条件要先判断 fast != null and fast.next != null，否则容易空指针。
    
3. 不要说“快指针在环里等慢指针”，准确说法是：快指针持续前进，并最终追上慢指针。
    

---

## 面试可复述版本

“这题我会先想到用 Hash Set 记录访问过的节点，如果某个节点再次出现，就说明链表有环。这是一个 O(n) 时间、O(n) 空间的解法。更优的方法是使用快慢指针，慢指针每次走一步，快指针每次走两步。如果链表有环，两者最终会在环内相遇；如果没有环，快指针会先走到 null。因此最优解时间复杂度是 O(n)，空间复杂度是 O(1)。”

# 1. goroutine 是什么

### 答法

goroutine 是 Go 里的轻量级并发执行单元。  
可以理解成比线程更轻，创建和切换成本更低，适合高并发服务端场景。

### 你要会说

- Go 天然支持高并发
    
- goroutine 适合处理大量并发任务
    
- 它不是传统意义上的操作系统线程
    

### 追问

- goroutine 和 thread 有什么区别？
    
- 为什么 Go 并发开销更小？

# 2. channel 是什么

### 答法

channel 是 goroutine 之间通信的一种机制，用来安全地传递数据。  
Go 更强调 通过通信共享数据，而不是直接共享内存。

### 你要会说

- channel 常用于 goroutine 协作
    
- 可以做任务传递、同步、生产者消费者模型
    

### 追问

- channel 有缓冲和无缓冲有什么区别？
    
- 关闭 channel 会怎样？
    

---

# 3. Go 为什么适合后端开发

### 答法

因为 Go 语法相对简洁，编译部署方便，并发模型轻量，比较适合开发高并发服务、网络服务和基础设施类系统。

### 你要会说

- 开发效率和运行效率比较平衡
    
- 部署简单
    
- 并发支持友好
    

### 追问

- Go 和 Java 各自适合什么场景？
    
- 为什么很多云原生项目喜欢 Go？
    

---

# 4. Go 和 Java 并发模型的大致区别

### 答法

Java 更偏基于线程、锁和共享内存的并发模型；  
Go 更强调 goroutine + channel 这种轻量协作方式。

### 你要会说

- Java 并发生态成熟
    
- Go 并发表达更轻
    
- Go 更容易写出高并发网络服务
    

### 追问

- goroutine 一定比 thread 好吗？
    
- Go 还需要锁吗？

# 5. slice 是什么

### 答法

slice 可以理解成对数组的一个动态封装，比数组更灵活，长度可变，是 Go 里很常用的数据结构。

### 你要会说

- 数组长度固定
    
- slice 更适合日常开发
    
- append 是高频操作
    

### 追问

- slice 扩容会发生什么？
    
- slice 和数组的区别是什么？

# 6. map 是什么

### 答法

map 是键值对结构，类似 Java 里的 HashMap，适合做快速查找。

### 你要会说

- 适合做索引和查表
    
- Go 的 map 不是线程安全的
    

### 追问

- 并发读写 map 会怎样？
    
- Go map 和 Java HashMap 有什么异同？

# 7. interface 是什么

### 答法

interface 可以理解成一组行为约定。  
只要一个类型实现了接口要求的方法，就算实现了这个接口，不需要显式声明。

### 你要会说

- Go 接口是隐式实现
    
- 接口用于解耦和抽象
    

### 追问

- Go 接口和 Java interface 有什么区别？
    
- 为什么说 Go 接口更轻？
    

---

# 8. error 处理方式

### 答法

Go 一般不用异常体系做主流程控制，而是通过函数返回值显式返回 error，再由调用方决定如何处理。

### 你要会说

- Go 更强调显式错误处理
    
- 代码会更直接，但也可能更啰嗦
    

### 追问

- Go 为什么不太像 Java 那样依赖异常？
    
- 这样设计有什么好处和代价？

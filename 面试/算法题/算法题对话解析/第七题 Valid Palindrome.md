# 题目：Valid Palindrome

给你一个字符串 s，判断它是否是 palindrome。

这里的规则是：

- 只保留 字母和数字（alphanumeric，字母数字字符）
    
- 忽略大小写（case-insensitive，不区分大小写）
    
- 然后判断正着读和反着读是否一样
    

### 示例 1

Input: s = "A man, a plan, a canal: Panama"

Output: true

  

解释：清洗后得到 "amanaplanacanalpanama"，正反一样。

### 示例 2

Input: s = "race a car"

Output: false

  

解释：清洗后得到 "raceacar"，正反不一样。

### 示例 3

Input: s = " "

Output: true

  

解释：清洗后是空串，空串按题目默认算回文。

---

这题大概是 字符串 + 双指针（two pointers） 的题型候选，本质是：一边跳过无效字符，一边从两端向中间比较。

3. 这题最核心的动作其实只有两个：
    

- 过滤无效字符
    
- 比较左右字符是否相等

- 遇到有效字符就比较

- baseline solution：先清洗出新字符串，再判断回文

## 核心术语

- Palindrome：回文，正着读和反着读一样的字符串。

- Alphanumeric：字母数字字符，也就是 [a-zA-Z0-9] 这一类有效字符。

## 先修正你的两道追问答案

### 追问 1 修正版

为什么 optimal 的时间复杂度是 O(n)，不是 O(n^2)？

面试可复述版本：

因为左右指针都只会朝中间单向移动，不会回退。虽然过程中可能会跳过无效字符，但每个字符最多被左指针或右指针访问常数次，所以总遍历次数与字符串长度成正比，时间复杂度是 O(n)。

你原来的“相加不是相乘”不算错，但不够本质。  
更本质的说法是：每个字符不会被反复来回扫描。

---

### 追问 2 修正版

如果不能忽略特殊字符，也不能忽略大小写，解法怎么变？

面试可复述版本：

仍然可以使用双指针。区别在于不需要跳过特殊字符，也不需要做大小写转换，直接比较原字符串左右两端字符是否完全相等即可。如果有一次不相等，就返回 false；否则继续向中间收缩。

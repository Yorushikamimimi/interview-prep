- 用一个滚动变量 suffix 动态维护右边信息
    

---

### 4. 最优解复杂度

设：

- n = 数组长度
    

那么：

#### Time Complexity

- 两次线性遍历
    
- 总体是 O(n)
    

#### Space Complexity

如果按题目的经典口径：

- 不算返回的 answer 数组
    
- 额外只用了 prefix 和 suffix 两个变量
    

所以：

- O(1) extra space
    

注意这里的术语：

- extra space：额外空间，不把题目要求返回的答案数组算进去
    

如果在特别严格、特别学院派的口径下，把返回数组也算空间，那会是 O(n)。  
但这道题标准面试 / LeetCode 口径通常说：

- O(1) extra space
    

---

### 5. baseline 和 optimal 的优化点

你面试里可以这样讲：

#### baseline

- 对每个位置都重新遍历整个数组
    
- 跳过自己，乘其他元素
    
- 时间复杂度 O(n²)
    

#### 中间优化

- 用 left 和 right 两个数组预处理
    
- 时间降到 O(n)
    
- 空间 O(n)
    

#### optimal

- 让 answer 先存左边乘积
    
- 再用一个滚动 suffix 补右边乘积
    
- 时间仍是 O(n)
    
- 额外空间降到 O(1)
    

---

### 6. 容易错的点

这题面试里最常错的就是这几个：

#### 错点 1：想用总乘积除自己

问题：

- 题目通常不允许除法
    
- 遇到 0 会很麻烦
    

#### 错点 2：空乘积写成 0

正确应该是：

- 空乘积 = 1
    

#### 错点 3：第二遍顺序写反

错误写法会把 nums[i] 自己也并进 suffix，导致结果不再是 except self

正确顺序是：

answer[i] *= suffix

suffix *= nums[i]

# 题目

Valid Parentheses

给定一个只包含 '('、')'、'{'、'}'、'['、']' 的字符串 s，判断字符串是否有效。

一个有效字符串需要满足：

1. 左括号必须用相同类型的右括号闭合
    
2. 左括号必须按正确顺序闭合
    
3. 每个右括号都必须有对应的左括号
    

### 示例

示例 1

输入: s = "()"

输出: true

  

示例 2

输入: s = "()[]{}"

输出: true

  

示例 3

输入: s = "(]"

输出: false

  

示例 4

输入: s = "([)]"

输出: false

  

示例 5

输入: s = "{[]}"

输出: true

  

---

## 先给结论

这题大概属于 Stack（栈）题 / 括号匹配题。  
本质是：判断“最近打开但还没关闭”的括号，能不能被当前右括号正确匹配。

---

## 已知事实

1. 这题不是在数括号个数，而是在看 匹配顺序。  
    比如 "([)]" 里每种括号数量都对，但顺序错了，所以仍然无效。
    
2. 这题很适合用 Stack（栈）。  
    因为栈是“后进先出”，正好对应“最后打开的括号，必须最先关闭”。
    
3. 遍历字符串时：
    

- 遇到左括号，先记下来
    
- 遇到右括号，必须去匹配“最近一个还没匹配的左括号”
    

5. 最后如果还有没被匹配掉的左括号，字符串也无效。  
    例如 "((("。
    

---

## 合理推测

1. baseline 思路你可能会想到“反复消掉 () / [] / {}”，这个能做，但不够自然，也可能更慢。
    
2. 这题的标准最优思路大概率就是 一次遍历 + 栈。
    
3. 真正常见的错误点不是“不会写代码”，而是：
    

- 没意识到这是“顺序匹配”问题
    
- 不知道为什么必须用栈而不是计数器
    
- 忘了处理“栈空但来了右括号”这种情况

# 最终总结答案

## 1）一句话题型总结

这题属于 栈题，本质是用栈维护“最近未匹配的左括号”，检查括号是否按正确顺序闭合。

## 2）一句话核心 insight

当前右括号只能匹配栈顶那个最近未闭合的左括号。

---

## 3）baseline solution（朴素解）

### Core Idea

不断删除字符串中的：

- ()
    
- []
    
- {}
    

直到不能再删除为止。  
如果最后字符串为空，则有效；否则无效。

### 为什么可行

因为合法括号串最终都能被逐步消掉。

### 为什么慢

- 需要反复扫描字符串
    
- 删除后可能产生新的可消对，要继续扫
    
- 删除还会带来字符串重建/数组移动成本
    

---

## 4）optimal solution（最优解）

### Core Idea

从左到右遍历字符串：

- 遇到左括号：push 进栈
    
- 遇到右括号：
    

- 如果栈空，返回 false
    
- 否则检查它是否和栈顶左括号匹配
    
- 匹配则 pop
    
- 不匹配则返回 false
    

- 遍历结束后：
    

- 栈空，返回 true
    
- 栈不空，返回 false
    

---

## 5）伪代码骨架

创建一个空栈 stack

  

遍历字符串 s 中的每个字符 c:

    如果 c 是左括号:

        push(c)

    否则 c 是右括号:

        如果 stack 为空:

            return false

        top = stack.top()

        如果 top 和 c 不匹配:

            return false

        stack.pop()

  

如果 stack 为空:

    return true

否则:

    return false

  

---

## 6）复杂度分析

设 n 为字符串长度。

### baseline

- Time Complexity: 通常可看作 O(n^2)
    
- Space Complexity: 视实现而定，常见为 O(n)
    

### optimal（栈解）

- Time Complexity: O(n)  
    每个字符最多进栈一次、出栈一次
    
- Space Complexity: O(n)  
    最坏情况下全是左括号，全部压栈
    

---

## 7）容易错的点

1. 不要只统计左右括号数量，顺序也很重要
    
2. 不是“右括号出栈”，而是“左括号在栈里，右括号来触发匹配”
    
3. 不是只判断第一个字符，任何位置只要右括号出现时栈空，都要 false
    
4. 最后一定要检查栈是否为空

## 8）面试可复述版本

我会使用一个栈来保存尚未匹配的左括号。遍历字符串时，遇到左括号就压栈；遇到右括号时，先判断栈是否为空，如果为空说明没有可匹配的左括号，直接返回 false。若栈不为空，则检查栈顶左括号是否和当前右括号匹配，若不匹配也返回 false，匹配则弹出栈顶。遍历结束后，如果栈为空，说明所有括号都成功匹配，返回 true，否则返回 false。

## 10）这题最终你该记住的三句话

1. 这是一道典型栈题，不是计数题。
    
2. 右括号只能匹配栈顶那个最近未闭合的左括号。
    
3. 一趟遍历，匹配失败立刻返回，最后看栈是否为空。

题目：Binary Tree Level Order Traversal

给定一棵二叉树的根节点 root，返回其节点值的层序遍历结果。也就是按“从上到下、从左到右”一层一层地收集节点。

示例：

输入：root = [3,9,20,null,null,15,7]

输出：[[3],[9,20],[15,7]]

  

对应的树大致是：

     3

     / \

    9  20

      /  \

     15   7

  

---

## 结论

这题大概率属于 Binary Tree Traversal 里的 BFS（Breadth-First Search，广度优先搜索） 题型候选，本质是：按层访问二叉树，并把每一层单独收集起来。

---

## 已知事实

1. 题目不是让你“遍历所有节点就行”，而是要求你按层分组输出。
    
2. 最常见的核心数据结构是 Queue（队列），因为它天然适合“先进先出”，很适合一层一层往外扩。
    
3. 这题最经典的做法是 BFS，因为 BFS 天然就是“层级扩散”。
    
4. 你最后返回的不是一维数组，而是 二维数组，例如 [[3],[9,20],[15,7]]，说明“每层都要单独存”。

3. 真正的难点不是“访问节点”，而是：怎么知道当前这一层有多少个节点，从而把它们单独放进一个数组里。

## 题目

Binary Tree Level Order Traversal

给定二叉树根节点 root，按层从上到下、从左到右返回节点值。

示例：

输入：root = [3,9,20,null,null,15,7]

输出：[[3],[9,20],[15,7]]

  

---

## 结论

这题属于 二叉树层序遍历 / BFS（Breadth-First Search，广度优先搜索），本质是：用队列按层推进，并把每一层单独收集成一个数组。

---

## 已知事实

1. 最合适的解法是 BFS + Queue（队列）。
    
2. 队列里放的是“还没处理的节点”。
    
3. 每一轮先记录当前队列长度 size，这个 size 就代表“当前层有多少个节点”。
    
4. 处理这一层时，把节点值加入 level，再把它们的左右孩子入队。
    
5. 当前层处理完后，把 level 放进结果数组 res。
    

---

## 合理推测

1. 这题的核心难点不是“怎么遍历树”，而是怎么把每一层分开。
    
2. 关键突破点就是：每层开始前先固定住当前层节点数 size。
    
3. DFS 也能做，但对于这题，BFS 更直观、更贴题意。
    

---

## 需要验证

1. 你是否已经真正理解：queue.size() 在“这一轮开始时”表示当前层大小。
    
2. 你是否能区分：
    

- 当前层节点：这一轮要弹出的 size 个节点
    
- 下一层节点：这些节点的孩子，入队但本轮不处理
    

---

## 核心术语

- BFS：一层一层向外扩展的遍历方式。
    
- Queue：先进先出，适合层级推进。
    
- Level Order Traversal：按层遍历二叉树。
    
- level size：当前层节点数量，通常等于本轮开始时的队列长度。
    

---

## 解题思路

### baseline solution

直接用 BFS + Queue：

1. 若 root == null，返回空数组。
    
2. 根节点入队。
    
3. 当队列不为空时：
    

- 记录当前层大小 size = queue.size()
    
- 创建当前层数组 level = []
    
- 循环 size 次：
    

- 弹出一个节点 node
    
- 把 node.val 放入 level
    
- 若左孩子非空，入队
    
- 若右孩子非空，入队
    

- 把 level 加入结果 res
    

5. 返回 res
    

---

## 伪代码骨架

if root == null:

    return []

  

queue = [root]

res = []

  

while queue is not empty:

    size = queue.size()

    level = []

  

    repeat size times:

        node = queue.pop_front()

        level.add(node.val)

  

        if node.left != null:

            queue.push_back(node.left)

        if node.right != null:

            queue.push_back(node.right)

  

    res.add(level)

  

return res

  

---

## 为什么这样设计

因为队列天然满足“先到先处理”，很适合树的层级访问。  
而“每层开始时先记录 size”，就能保证：

- 本轮只处理当前层
    
- 新加入队列的孩子节点留到下一层处理
    

这就是“按层分组”的关键。

---

## 容易错的点

1. 忘记判空：root == null 时应返回 []。
    
2. 没有先固定 size：会导致当前层和下一层混在一起。
    
3. 把已处理节点还留在队列里理解错了：节点一旦出队，就已经处理完了。
    
4. 结果数组层级错乱：每层都要新建一个 level，不能复用同一个数组。
    

---

## 复杂度分析

设二叉树总节点数为 n。

- 时间复杂度：O(n)  
    每个节点只会入队、出队各一次。
    
- 空间复杂度：O(n)  
    最坏情况下队列里可能存一整层节点，数量级可到 n。
    

---

## 面试可复述版本

这题是标准的二叉树层序遍历，优先想到 BFS。做法是用队列保存待处理节点，每一轮先记录当前队列长度作为当前层节点数，再循环处理这一层的所有节点，并把它们的左右孩子加入队列。这样就能自然得到按层分组的结果。时间复杂度 O(n)，空间复杂度 O(n)。

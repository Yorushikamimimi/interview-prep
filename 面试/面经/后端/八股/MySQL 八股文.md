## 1. 索引

**答:** 帮助数据库快速查找数据的数据结构，用空间换时间。

### 为什么用 B+Tree

- 层数低、磁盘 IO 少
- 叶子节点有序链接，天然适合范围查询
- 比二叉树/红黑树层数更低，IO 更少

### 聚簇索引 vs 非聚簇索引

- **聚簇索引:** 索引和数据放一起，InnoDB 主键索引，叶子存整行数据
- **非聚簇索引:** 叶子存主键值，查完还要**回表**到聚簇索引取完整数据

### 覆盖索引

- 查询字段都在索引里 → 直接返回，不需要回表

### 最左前缀原则

- 联合索引 (a,b,c) 要从最左列开始匹配
- 范围查询后的列索引利用可能下降

### 索引失效常见场景

- 对索引列做函数/计算/隐式类型转换
- 不满足最左前缀
- `LIKE '%abc'` 前导通配符
- `OR` 使用不当
- 索引区分度太低，优化器放弃

## 2. 事务

### ACID

| 特性 | 含义 |
|------|------|
| 原子性 | 要么全成功，要么全失败 |
| 一致性 | 事务前后数据满足业务规则 |
| 隔离性 | 多事务互不干扰 |
| 持久性 | 提交后结果持久保存 |

### 隔离级别（由低到高）

| 级别 | 脏读 | 不可重复读 | 幻读 |
|------|------|-----------|------|
| Read Uncommitted | 有 | 有 | 有 |
| Read Committed | 无 | 有 | 有 |
| Repeatable Read (MySQL默认) | 无 | 无 | 有(InnoDB部分解决) |
| Serializable | 无 | 无 | 无 |

### 脏读 / 不可重复读 / 幻读

- **脏读:** 读到未提交数据
- **不可重复读:** 同一行前后读取结果不一致（被别人改了）
- **幻读:** 同一范围查询前后行数变了（被别人插了）

## 3. MVCC

**答:** 多版本并发控制，让读写更少互相阻塞。普通 SELECT 读的是一致性快照，不阻塞写。

- 依赖 undo log（旧版本）+ Read View（事务可见性判断）

### 三种日志

| 日志 | 作用 |
|------|------|
| redo log | 崩溃恢复，保证持久性 |
| undo log | 事务回滚 + MVCC 多版本 |
| binlog | 归档日志，主从复制 |

## 4. InnoDB vs MyISAM

| 对比 | InnoDB | MyISAM |
|------|--------|--------|
| 事务 | 支持 | 不支持 |
| 锁 | 行锁 | 表锁 |
| 外键 | 支持 | 不支持 |
| 场景 | 高并发 OLTP | 读多写少(已少用) |

## 5. 慢查询排查

**答:** 定位 SQL → EXPLAIN 看执行计划 → 检查索引 → 看 SQL 写法和数据量。

### EXPLAIN 重点字段

- **type:** 访问方式（越高效越好）
- **key:** 实际用到的索引
- **rows:** 估算扫描行数
- **Extra:** Using filesort / Using temporary 等额外开销

## 6. 深分页

**答:** `LIMIT 100000, 10` 慢的原因是数据库要先扫描并丢弃前 10 万行。

优化方案：
- 游标分页：记住上一页最后 ID，`WHERE id > last_id LIMIT 10`
- 先查 ID 再回表：`SELECT * FROM t WHERE id IN (SELECT id ... LIMIT)`

### 2. 这题最重要的候选思路

#### 方案 A：排序 / Sorting

### 3. 关键术语解释

#### Hash Set

哈希集合。

#### Consecutive

连续的。  
比如 5, 6, 7, 8 是 consecutive。

#### Sequence start

序列起点。  
如果一个数 x 存在，但 x - 1 不存在，那么 x 就可能是一段连续序列的起点。

例如集合里有 {1,2,3,4,100}：

### 3. 正确理解应该是什么

#### 对于 [1,2,2,3]

先放进 Hash Set：

set = {1,2,3}

然后遍历这些数时：

- 1 的前驱 0 不存在，所以 1 是起点
    
- 从 1 开始往后找：2 在，3 在，所以长度是 3
    

而：

- 2 的前驱 1 存在，所以 2 不是起点，跳过
    
- 3 的前驱 2 存在，所以 3 不是起点，跳过
    

答案仍然是 3。

### 2. 你这次的两个概念错误

#### 错误 A：set 为什么快

你说：

set 查询快的原因是避免了排序

这不准确。

更准确的说法是：

set 快，是因为哈希结构支持平均 O(1) 的 membership check。

也就是：

- 我想知道 x 在不在
    
- 我想知道 x+1 在不在
    
- 我想知道 x-1 在不在
    

这些查询平均都很快。

“避免排序”只是结果，不是原因。

---

#### 错误 B：起点不是最大值

你说：

从起点开始扫描，就已经是该段序列的最大值了

这句反了。

例如序列：

1,2,3,4

- 起点是 1
    
- 不是最大值
    
- 它是这段连续序列的 最小值 / 开头
    

更准确地说：

从起点开始扫描，可以把整段连续序列完整覆盖到终点。

### 3. 为什么时间复杂度是 O(n) 平均

你可以这样理解：

#### 第一步：建 set

把所有元素放入 set：

- 时间复杂度平均是 O(n)
    

#### 第二步：遍历每个数，判断是不是起点

对每个数做一次：

- 看 x-1 在不在
    

这个判断平均是 O(1)，  
n 个数总共就是平均 O(n)。

#### 第三步：只从起点往后扩展

这一步看起来像可能会很多次循环，但关键在于：

每个数最多只会在某一条连续序列的扩展过程中被“真正往后扫描到”一次。

例如 {1,2,3,4}：

- 从 1 出发，会扫到 2,3,4
    
- 2,3,4 自己都不是起点，不会再各自重扫一遍
    

所以整段的扩展总成本仍然是线性的。

# JVM

---

## 1. JVM 内存区域

**答:** 主要分为堆、Java 虚拟机栈、本地方法栈、方法区、程序计数器。

- **堆:** 线程共享，存对象实例和数组，GC 主要区域
- **栈:** 线程私有，存局部变量、方法调用栈帧
- **方法区:** 存类信息、常量、静态变量（JDK 8+ 改为元空间 Metaspace，使用本地内存）

---

## 2. 堆 vs 栈

**答:** 栈存局部变量和方法调用信息（线程私有）；堆存对象实例（线程共享，需 GC）。

- 栈快：分配回收简单，方法结束自动弹出
- 堆需要 GC：对象生命周期不确定
- 注意：对象在堆里，引用变量可能在栈里

---

## 3. 对象创建过程

**答:** 类加载检查 → 分配内存 → 初始化零值 → 设置对象头 → 执行构造方法。

---

## 4. 垃圾回收

**答:** 主要回收堆。通过**可达性分析**判断对象是否可回收（从 GC Roots 出发搜索，不可达即为垃圾）。

- 不用引用计数：因为循环引用会失效

### 常见 GC 算法

| 算法 | 特点 |
|------|------|
| 标记-清除 | 简单，但有内存碎片 |
| 复制 | 适合存活对象少的场景 |
| 标记-整理 | 减少碎片，存活对象移到一端 |

### Minor GC vs Full GC

- Minor GC：回收新生代，频率高，成本小
- Full GC：涉及老年代，暂停时间长，尽量避免

---

## 5. 类加载

**答:** 加载 → 验证 → 准备 → 解析 → 初始化。

### 双亲委派

- 类加载器收到请求后先让父加载器尝试加载，父加载器加载不了自己再加载
- 作用：避免类重复加载 + 保护核心类库安全
- 三层：Bootstrap → Extension/Platform → Application
- 可通过自定义类加载器打破（SPI、热部署等场景）

---

## 6. 常见垃圾回收器

| 回收器 | 特点 |
|--------|------|
| Serial | 单线程，简单 |
| CMS | 追求低停顿，但有内存碎片 |
| G1 | 按 Region 划分堆，大堆场景更均衡 |

一句话：CMS 低停顿但有碎片，G1 更均衡更现代。

---

## 7. OOM vs StackOverflow

**答:** OOM 是内存不够（堆不足、元空间不足）；StackOverflow 是栈太深（无限递归）。

- OOM 常见原因：对象创建过多、内存泄漏、缓存无限增长
- StackOverflow 常见原因：递归无终止条件、调用链过深

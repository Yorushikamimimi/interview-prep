## 1. 题目和例子

- 题目：给你一个整数数组 `nums` ，求其中 **最长严格递增子序列** 的长度。
    
- 子序列：可以删元素，但不能改变剩余元素相对顺序。
    

示例 1：

- 输入：`nums = [10,9,2,5,3,7,101,18]`
    
- 输出：`4`
    

示例 2：

- 输入：`nums = [0,1,0,3,2,3]`
    
- 输出：`4`
    

示例 3：

- 输入：`nums = [7,7,7,7,7,7,7]`
    
- 输出：`1`
    

补充小例子：

- `nums = [4,10,4,3,8,9]`，输出 `3`
    
- 因为最长递增子序列可以是 `[4,8,9]`
    

---

## 2. 核心思路

- baseline：`dp[i]` 表示 **以 `nums[i]` 结尾的 LIS 长度**，时间复杂度 `O(n^2)`。
    
- optimal：用 **贪心 + 二分**，时间复杂度 `O(n log n)`。
    

### 最核心的方法：贪心 + 二分

维护一个数组 `tails`：

- `tails[k]` 表示：长度为 `k+1` 的递增子序列中，**结尾最小** 的那个尾值。
    

为什么这样做：

- 同样长度的递增子序列，结尾越小，后面越容易接更大的数。
    
- 所以我们总是希望尾巴尽量小。
    

处理每个数 `x`：

- 在 `tails` 中二分找到第一个 `>= x` 的位置
    
- 用 `x` 替换它
    
- 如果 `x` 比所有尾值都大，就直接加到后面
    

注意：

- `tails` 里的值不一定是真实的某个 LIS
    
- 但它的长度一定是 LIS 的长度
    

### 简短例子

`nums = [10,9,2,5,3,7,101,18]`

过程：

- 10 -> `[10]`
    
- 9 -> `[9]`
    
- 2 -> `[2]`
    
- 5 -> `[2,5]`
    
- 3 -> `[2,3]`
    
- 7 -> `[2,3,7]`
    
- 101 -> `[2,3,7,101]`
    
- 18 -> `[2,3,7,18]`
    

最终长度是 `4`

## 3. ACM 题解

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        List<Integer> list = new ArrayList<>();
        while (sc.hasNextInt()) {
            list.add(sc.nextInt());
        }

        int n = list.size();
        if (n == 0) {
            System.out.println(0);
            return;
        }

        int[] tails = new int[n];
        int len = 0;

        for (int x : list) {
            int left = 0, right = len;
            while (left < right) {
                int mid = left + (right - left) / 2;
                if (tails[mid] >= x) {
                    right = mid;
                } else {
                    left = mid + 1;
                }
            }
            tails[left] = x;
            if (left == len) {
                len++;
            }
        }

        System.out.println(len);
    }
}
```

## 4. LeetCode 提交版

```java
class Solution {
    public int lengthOfLIS(int[] nums) {
        int n = nums.length;
        int[] tails = new int[n];
        int len = 0;

        for (int x : nums) {
            int left = 0, right = len;
            while (left < right) {
                int mid = left + (right - left) / 2;
                if (tails[mid] >= x) {
                    right = mid;
                } else {
                    left = mid + 1;
                }
            }
            tails[left] = x;
            if (left == len) {
                len++;
            }
        }

        return len;
    }
}
```

## 5. 一句话口诀

- **结尾越小越好，二分替换找位置。**
    

---

## 6. 易错点

1. 这是 **严格递增**，所以二分找的是第一个 `>= x` 的位置，不是 `> x`。
    
2. `tails` 不是最终答案序列，只是帮助我们求长度。
    
3. 子序列不是子数组，不要求连续。

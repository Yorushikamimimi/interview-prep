题目：Daily Temperatures（每日温度）

给你一个整数数组 temperatures，其中 temperatures[i] 表示第 i 天的气温。  
请返回一个数组 answer，其中 answer[i] 表示：要等几天才会有更高的气温。  
如果之后都不会升高，就填 0。

示例：

输入: temperatures = [73,74,75,71,69,72,76,73]

输出: [1,1,4,2,1,1,0,0]

  

解释：

- 第 0 天 73，下一天 74 更高，所以是 1
    
- 第 2 天 75，要等到第 6 天 76，所以是 4
    
- 第 6 天 76，后面没有更高温度，所以是 0
    

---

## 先给结论

这题大概是 单调栈（Monotonic Stack） 题型候选，本质是：  
对于每个位置，快速找到“右边第一个比它更大”的元素。

    这题不是在求“后面有没有更大的数”，而是在求：  
    右侧第一个更高温度出现的位置距离。
    
2. baseline 思路

    这个思路能做出来，但会比较慢。

4. 为什么单调栈适合  
    因为它能帮我们把“重复向右找”的过程压缩掉，避免很多重复比较。

## 必须知道的核心术语

- Monotonic Stack（单调栈）：一种让栈内元素保持单调递增或递减的数据结构，常用于快速找到某个元素左边/右边第一个更大或更小的元素。
    
- Next Greater Element（下一个更大元素）：对于当前位置，找到右边第一个比它大的元素。
    
- Index（下标）：数组中的位置编号，这题最后要求的是“间隔天数”，所以通常不能只存值。

## 先给结论

这题属于 单调栈（Monotonic Stack） 经典题。  
本质是：对每个位置，找右边第一个比它大的元素（Next Greater Element）。

- baseline：双重遍历，逐个向右找第一个更高温度
    
- optimal：用单调递减栈存下标，一次遍历完成

## 已知事实

### 1. 题目本质

给定 temperatures[i]，要求返回：

- 对于每一天 i
    
- 右边第一个温度更高的位置 j
    
- 结果是 j - i
    
- 如果不存在，结果为 0
    

所以这题求的不是：

- 最大温度
    
- 后面有没有更高
    
- 所有更高温度
    

而是：

右边第一个更高温度的位置差

---

### 2. baseline solution（朴素解）

思路：

- 枚举每一天 i
    
- 再从 i+1 开始向右找
    
- 找到第一个满足 temperatures[j] > temperatures[i] 的位置
    
- 记录 answer[i] = j - i
    
- 如果一直没找到，就记 0
    

#### baseline 为什么会慢

因为存在大量重复扫描：

例如某一天往右看了很多元素，下一天又会重新看其中很多元素。  
也就是说，很多位置会被反复比较多次。

#### baseline 复杂度

- 时间复杂度：O(n^2)
    
- 空间复杂度：O(1)（不算答案数组）
    

其中：

- n = 温度数组长度
    

---

### 3. optimal solution（最优解）

用 单调栈（Monotonic Stack）。

#### 栈里放什么

栈里放的是：下标 index，不是温度值。

因为最后要算：

j - i

  

只存温度值没法直接算间隔天数。

#### 栈保持什么性质

维护一个单调递减栈：

- 栈底到栈顶对应的温度是递减的
    
- 也可以理解为：栈顶是“最近一个还没找到更高温度的下标”
    

#### 主流程

从左到右遍历每一天 i：

1. 当前温度 temperatures[i]
    
2. 只要栈不空，且当前温度大于栈顶下标对应的温度
    
3. 说明：当前这一天，就是栈顶那一天右边第一个更高温度
    
4. 弹出栈顶下标 idx
    

记录：  
answer[idx] = i - idx

5.   
    
6. 重复这个过程，直到当前温度不再大于栈顶温度
    
7. 把当前下标 i 入栈
    

遍历结束后，栈里剩下的下标，说明右边没有更高温度，答案保持 0

---

### 4. 为什么单调栈能优化

核心原因：

每个下标最多入栈一次、出栈一次。

所以虽然有 while，但总弹栈次数不会超过 n 次。  
这就把很多重复扫描压缩掉了。

#### optimal 复杂度

- 时间复杂度：O(n)
    
- 空间复杂度：O(n)

## 面试可复述版本

这题本质是求每个位置右边第一个更大元素的位置差，属于单调栈经典题。  
baseline 做法是对每个位置向右逐个查找第一个更高温度，时间复杂度是 O(n^2)。  
更优做法是维护一个存下标的单调递减栈，遍历数组时，如果当前温度比栈顶下标对应的温度高，就说明当前元素是栈顶元素右边第一个更大值，可以弹栈并计算答案 i - idx。  
每个元素最多入栈出栈各一次，因此总时间复杂度是 O(n)，空间复杂度是 O(n)。

---

## 伪代码骨架

initialize answer = [0] * n

initialize stack = empty   // 栈里存下标

  

for i in range(0, n):

    while stack is not empty and temperatures[i] > temperatures[stack.top]:

        idx = stack.pop()

        answer[idx] = i - idx

    stack.push(i)

  

return answer

  

---

## 容易错的点

1. 栈里存值而不是下标  
    错，因为最后要算天数差
    
2. 单调方向搞反  
    这题是维护“温度递减”的栈
    
3. 比较条件写成 >=  
    题目要求是“更高温度”，所以应是 >
    
4. 以为 while 会退化成 O(n^2)  
    不会，因为每个元素最多弹出一次

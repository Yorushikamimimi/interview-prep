# 计算机网络

---

## 1. TCP vs UDP

**答:** TCP 面向连接、可靠传输；UDP 无连接、不保证可靠但更快。

| 对比 | TCP | UDP |
|------|-----|-----|
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 确认应答+重传+流控+拥塞控制 | 不保证 |
| 场景 | HTTP、MySQL、Redis | 音视频、直播、DNS |

一句话：TCP 求稳，UDP 求快。

---

## 2. 三次握手

**答:** 确认双方收发能力正常，建立连接。

1. 客户端发 SYN
2. 服务端回 SYN + ACK
3. 客户端回 ACK

- 为什么不是两次：两次无法让服务端确认客户端的接收能力

---

## 3. 四次挥手

**答:** TCP 全双工，双方关闭要分开确认。

1. 一方发 FIN
2. 对方回 ACK
3. 对方处理完再发 FIN
4. 最后回 ACK

- 为什么不是三次：收到 FIN 的一方可能还有数据没发完，ACK 和 FIN 不能合并

---

## 4. HTTP vs HTTPS

**答:** HTTP 明文传输，HTTPS = HTTP + TLS/SSL，加密 + 证书身份校验 + 防篡改。

---

## 5. GET vs POST

**答:** 语义不同：GET 偏查询，POST 偏提交。

- GET 参数在 URL，POST 在请求体
- 安全性取决于 HTTP 还是 HTTPS，不是 GET/POST 本身
- GET 参数更易暴露在 URL/日志/浏览器历史中

---

## 6. Cookie / Session / Token

**答:** 三者不在同一层面：Cookie 是浏览器存储载体，Session 是服务端会话状态，Token 是客户端持有的认证凭证。

| 对比 | Cookie | Session | Token |
|------|--------|---------|-------|
| 存储 | 浏览器 | 服务端 | 客户端 |
| 适用 | 传统 Web | 传统登录 | 前后端分离/分布式 |

- 前后端分离更常用 Token：无状态、利于扩展、多端接入

---

## 7. 输入 URL 到页面返回

**答:** DNS 解析 → TCP 连接 → (HTTPS 则 TLS 握手) → 发送 HTTP 请求 → 服务端处理响应 → 浏览器解析渲染

---

## 8. HTTP 常见状态码

| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 301/302 | 永久/临时重定向 |
| 304 | 未修改，走缓存 |
| 400 | 请求参数错误 |
| 401 | 未认证（没登录） |
| 403 | 已认证但无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |
| 502/504 | 网关错误/超时 |

---

## 9. HTTP/1.1 vs HTTP/2

**答:** HTTP/2 核心改进：多路复用、头部压缩、二进制分帧，让一个连接更高效。

---

## 10. 长连接 vs 短连接

**答:** 短连接一次请求后断开，长连接可复用多次请求。HTTP/1.1 默认长连接。

---

## 高频对比速记

| 对比 | 核心区别 |
|------|----------|
| TCP vs UDP | 可靠面向连接 vs 不可靠无连接 |
| 三次握手 vs 四次挥手 | 建连确认收发 vs 断连分开关闭 |
| HTTP vs HTTPS | 明文 vs 加密+证书 |
| GET vs POST | 查询语义 vs 提交语义 |
| Cookie vs Session vs Token | 浏览器存储 / 服务端状态 / 客户端凭证 |
| 401 vs 403 | 未认证 vs 无权限 |
| HTTP/1.1 vs HTTP/2 | 队头阻塞 vs 多路复用 |
